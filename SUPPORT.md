# Support

Need help with ThreatGuard? Here's where to go.

## Asking questions

For "how do I…", "is this expected?", or general usage questions, open a
**GitHub Discussion** (the **Discussions** tab). Discussions keep answers
searchable for the next person. Please search existing discussions first.

Before asking, it usually helps to check:

- The [README](README.md) — overview, setup, configuration.
- [FAQ.md](FAQ.md) — the most common questions (LLMs, local models, accuracy,
  data handling, self-hosting).
- [ARCHITECTURE.md](ARCHITECTURE.md) — how the system fits together.
- [KNOWN_LIMITATIONS.md](KNOWN_LIMITATIONS.md) — what the tool does and does not
  do today.

## Reporting a bug

Open a **GitHub Issue** using the **Bug report** template. A good report
includes:

- What you expected vs. what happened.
- Exact steps to reproduce.
- Your environment (OS, Python version, LLM provider or rules-only).
- Relevant logs or error output (redact any secrets).

## Requesting a feature

Open a **GitHub Issue** with the **Feature request** template, or start a
**Discussion** if you'd like to shape the idea first. Explain the problem you're
trying to solve, not just the solution — it helps us find the best fit. Check the
[ROADMAP.md](ROADMAP.md) to see if it's already planned.

## Reporting a security vulnerability

**Do not** use public issues for security problems. Follow the private process in
[SECURITY.md](SECURITY.md) (GitHub → Security → Report a vulnerability).

## Discussion guidelines

- Be respectful and constructive — the [Code of Conduct](CODE_OF_CONDUCT.md)
  applies everywhere.
- Keep threads on-topic; open a new one for a new question.
- Share reproducible details rather than screenshots of text where possible.
- Remember this is a community project maintained on a best-effort basis —
  patience and clear write-ups get faster, better answers.

## Response expectations

This is an open-source project maintained on a volunteer, best-effort basis.
There is no commercial SLA. Clear, reproducible reports and well-scoped questions
are the fastest path to a good answer.
