# ThreatGuard v0.1.1 — Patch Release

A single bug-fix release addressing a stability regression found in v0.1.0
acceptance testing, before public announcement. No other changes.

## 🐛 Fixed

**CSV export crashed (HTTP 500) for system names with non-latin-1 characters.**

The CSV export placed the raw system name directly into the
`Content-Disposition` filename. HTTP headers are latin-1, so a system name
containing an em dash, en dash, accented letter, CJK text, or emoji raised a
`UnicodeEncodeError` and returned a 500 — including the shipped
`retail-platform.json` example (*"ShopFast — Online Retail Platform"*).

The filename is now built per **RFC 6266 / RFC 5987**: an ASCII-safe `filename`
fallback plus a UTF-8 `filename*` parameter that preserves the original Unicode
name. Only the header changed; CSV content is unchanged and remains UTF-8.

Other export formats (HTML, PDF, Markdown) were not affected.

## ✅ Validation

- New regression suite (`tests/test_csv_filename.py`) covers **em dash, en dash,
  accented, CJK, and emoji** names across both CSV export paths — 41 checks.
- Full test suite: **8/8** passing; `ruff` clean; CI green.
- Live verification: the `retail-platform` example now exports CSV successfully
  (HTTP 200); HTML/PDF/Markdown exports and the CLI continue to work.

## Upgrade

No action required beyond pulling `v0.1.1`. No configuration, dependency, or data
changes.
