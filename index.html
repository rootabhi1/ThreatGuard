How it works

From a description to a reviewed threat model

A deterministic rule engine does the modeling; you refine it; AI is optional enrichment on top. Every output is a draft for a human to review — not a security sign-off.

01 · DESCRIBE
Text, structured, or diagram
Type a plain-English description, enter a precise structured spec, or upload a PNG/JPEG/WebP architecture diagram. Components, flows, and trust boundaries are extracted automatically.
02 · ANALYZE
Rules + optional AI
STRIDE / PASTA / LINDDUN run deterministically with DREAD scoring and CVSS / CWE / ATT&CK / OWASP / compliance mapping. A configured LLM can add context-specific threats on top.
03 · REFINE
DFD editor + security attributes
Drag the data-flow diagram, edit components and flows, and answer Microsoft-TMT-style security properties. Re-run to fold your answers into a sharper threat list.
04 · TRACK & SHARE
Reports, status, diffs
Track per-threat remediation status, diff releases, export HTML / PDF / Markdown / CSV / executive reports, share read-only links, and file Jira tickets.

Quick start

Run locally in 4 steps

Python 3.11+. Or skip to docker compose up --build.

1
Clone the repo
The app lives in threat-modeler/
git clone https://github.com/rootabhi1/ThreatGuard
cd ThreatGuard/threat-modeler
2
Virtual env + install
Isolate dependencies
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
3
Set environment variables
Only JWT_SECRET is required
# Required
export JWT_SECRET=$(python3 -c "import secrets;print(secrets.token_urlsafe(48))")

# Optional — seed an admin (else the first person to register becomes admin)
export INITIAL_ADMIN_EMAIL=admin@example.com
export INITIAL_ADMIN_PASSWORD=ChangeMe123!

# Optional — enable AI (or set it later in Admin → Settings)
export ANTHROPIC_API_KEY=sk-ant-...   # or OPENAI_API_KEY (+ OPENAI_BASE_URL)
4
Start the server
Then open http://localhost:8000
# any of:
./run.sh
uvicorn app:app --reload --port 8000
docker compose up --build
✓ Verify: curl http://localhost:8000/healthz → {"status":"ok"}.  Interactive API explorer at /docs.

Troubleshooting

Common issues

Most problems are solved by setting JWT_SECRET and activating the venv.

🟢AI features show "not configured"▼

Expected without a key — the rule engine still produces a full threat model. Add a key in Admin → Settings → AI provider, or set an env var, to enable AI threat suggestions, diagram vision, executive narratives, and code fixes.

export ANTHROPIC_API_KEY=sk-ant-...   # or OPENAI_API_KEY
🔴ModuleNotFoundError on startup▼

The venv isn't activated or dependencies aren't installed. You should see (.venv) in your prompt.

source .venv/bin/activate
pip install -r requirements.txt
🔴Port 8000 already in use▼
uvicorn app:app --reload --port 8001
# or free 8000: lsof -ti:8000 | xargs kill
🔴401 Unauthorized on API calls▼

Token expired or the server restarted without a persistent JWT_SECRET. Re-login for a fresh token, and set a permanent JWT_SECRET so tokens survive restarts.

💡Interactive API docs (Swagger)▼

FastAPI auto-generates docs at /docs — click Authorize, paste your bearer token, and try any endpoint without writing curl.
