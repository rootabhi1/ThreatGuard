Features

Everything a security team needs

From small startups to enterprise teams — all in one self-hosted tool.

📝
Text-to-Threat-Model
Paste any plain-English system description. The engine auto-extracts components, data flows, and trust boundaries — no diagram drawing required.
🔍
4 Methodologies in Parallel
Run STRIDE, DREAD, PASTA, and LINDDUN simultaneously on the same system. Each methodology surfaces different classes of threats.
✦
Claude AI Enhancement
Optional — connect your Anthropic API key for context-specific threat reasoning, smarter trust boundaries, and richer mitigation strategies.
👥
Full RBAC
Admin, Management, and User roles. Feature-level access control. Admins manage users and releases. Users work within assigned features.
📊
Auto-Generated DFDs
SVG data flow diagrams auto-rendered from your extracted components. Shows Internet / DMZ / App / Data trust zone boundaries.
📄
Export Reports
One-click export to PDF (for stakeholders), HTML (interactive), or Markdown (version-controlled docs, Confluence). Also exportable via REST API.
🔐
JWT Auth + Audit Log
Access and refresh tokens with secure rotation. Every action — logins, status changes, user role updates — written to a tamper-evident audit log.
🐳
Docker Ready
Multi-stage Dockerfile using python:3.12-slim (~160MB). SQLite persisted via volume mount. Health check endpoint included. One command to run.
☁️
Azure Deployable
Bicep infrastructure-as-code templates + GitHub Actions CI/CD pipeline included. Deploy to Azure App Service with one script. ~$18/month.
Methodologies

Industry-standard threat frameworks

Run any combination simultaneously — each surface different risk dimensions.

STRIDE
Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, Elevation of Privilege

  • Data integrity
  • Accountability gaps
  • Data exposure
  • Availability attacks
  • Privilege abuse
DREAD
Damage, Reproducibility, Exploitability, Affected Users, Discoverability
  • 0–10 risk scoring
  • Prioritisation aid
  • Stakeholder comms

  • Per-threat scores
PASTA

  • Attacker-centric
  • Business risk focus
  • 7-stage process
  • Attack tree analysis
  • Risk-based output
  • Compliance-ready
LINDDUN
Linkability, Identifiability, Non-repudiation, Detectability, Disclosure, Unawareness, Non-compliance
  • Privacy-focused
  • GDPR alignment

How it works

From description to threat model in 4 steps

1
Describe Your System
Write a plain-English description of your architecture — components, data flows, users, and any sensitive data handled.
2
Engine Extracts Components
The text extractor identifies services, databases, APIs, users, and auto-assigns them to trust zones (Internet / DMZ / App / Data).
3
Threats Are Generated
STRIDE/DREAD/PASTA/LINDDUN rules fire against each component and data flow. Claude AI optionally adds context-specific threats.
4
Review, Track & Export
Triage threats by severity, update statuses, view the auto-generated DFD, and export a full report as PDF, HTML, or Markdown.
Installation

Three ways to run ThreatGuard

# 1. Clone and extract
git clone https://github.com/rootabhi1/Automated-Threat-Modelling.git
cd Automated-Threat-Modelling
unzip threat-modeler.zip && cd threat-modeler

# 2. Set environment variables
cp .env.example .env
# Edit .env: set INITIAL_ADMIN_EMAIL, INITIAL_ADMIN_PASSWORD, JWT_SECRET
# Generate JWT_SECRET:
python3 -c "import secrets; print(secrets.token_urlsafe(48))"

# 3. Build and run
docker compose up --build

# Open → http://localhost:8000
# Health check → http://localhost:8000/api/health
git clone https://github.com/rootabhi1/Automated-Threat-Modelling.git
cd Automated-Threat-Modelling && unzip threat-modeler.zip && cd threat-modeler

# Create and activate virtual environment
python3 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt

# Set required environment variables
export INITIAL_ADMIN_EMAIL=admin@example.com

export JWT_SECRET=$(python3 -c "import secrets; print(secrets.token_urlsafe(48))")
export ANTHROPIC_API_KEY=sk-ant-...  # optional — enables AI enrichment

# Start the server
python app.py
# Open → http://127.0.0.1:8000

API Reference

REST API

Full interactive Swagger docs available at /docs when the server is running.

MethodEndpointAuthDescription
POST/api/auth/loginPublicLogin → returns access + refresh tokens
POST/api/auth/registerPublicSelf-register (User role by default)
POST/api/threat-models🔐 JWTCreate a new threat model
POST/api/threat-models/{id}/analyze🔐 JWTRun full analysis (STRIDE/DREAD/PASTA/LINDDUN + AI)
GET/api/threat-models/{id}/report/{fmt}🔐 JWTExport report — fmt: pdf | html | md
PUT/api/threat-models/{id}/threats/{tid}/status🔐 JWTUpdate threat status (open / mitigated / accepted)
POST/api/extract-from-textPublicExtract system components from plain text
GET/api/releases🔐 JWTList all releases
POST/api/releases🔐 AdminCreate a release
GET/api/audit-log🔐 AdminFull audit log of all actions
GET/api/healthPublicHealth check — returns LLM availability status
